Instalação Padrão do playwright
npm init playwright@latest

Nosso projeto utilizará as seguintes abordagens:
- Separação dos cenários de negócio dos scripts de automação
 - vamos precisar de no mímino 2 arquivos
 ---> pasta features, que contém arquivos .feature - Funcionalidades e Cenários de negócios  / Critérios de Aceitação / Dados
 ---> pasta steps, que contém arquivos .spec.js - Script de automação de teste
    Instalações adicionais:
    // Instala o pacote que dá suporte ao bdd
    // BDD = Desenvolvimento Direcionado por Comportamento
    npm install -D playwright-bdd
    

    feature: Login
        Scenario: Login with Sucess
            Given
            When
            Then

    Funcionalidades: Login
        Cenario: Login com Sucesso
            Dado
            Quando
            Entao     

Nomeclatura
- BDD - Behavir Driven Development - Desenvolvimento Direcionado pelo Comportamento
  Abordagem de Desenvolvimento, muito valorizada dento da Agilidade
  Objetivo é facilitar a comunicação entre os menbros do time, áreas e clientes

- Gherkin: as instruções do BDD são escritas na sintaxe (formato) chamado Gherkin

- Cucumber/Behave/SpecFlow são exemplos de plugins que fazem a integração entre arquivos .features 
com arquivos .spec.js
